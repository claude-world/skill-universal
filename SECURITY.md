# Security Policy

## Supported Versions

Currently, only the latest version of Skill Universal is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, send an email to: **security@claude-world.com**

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What Happens Next

1. We will acknowledge receipt of your report within 48 hours
2. We will investigate the vulnerability
3. We will provide a timeline for the fix
4. We will notify you when the fix is released
5. We will publicly disclose the vulnerability after the fix is released

### Security Best Practices

When using Skill Universal:

- Keep dependencies updated
- Review skill code before loading untrusted skills
- Validate skill metadata before execution
- Use in sandboxed environments when possible
- Follow principle of least privilege

### Dependencies

We regularly update dependencies to address security vulnerabilities. If you
discover a security issue in our dependencies, please report it following the
process above.

### Security Audits

This project has not yet undergone a professional security audit. We welcome
security researchers to review our code and report vulnerabilities responsibly.
